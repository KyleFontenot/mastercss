## What is Master CSS? [sr-only]

### A CSS language
Master CSS is a [markup-driven](/docs/markup-driven) CSS language with smart rules allowing you to write familiar CSS using concise syntax within HTML.

Under the hood, Master CSS automatically adds [vendor prefixes](/docs/browser-support), smartly [ordering CSS rules](/docs/markup-driven#smart-rule-ordering), and more. This allows you to focus on building your website rather than spending time writing and maintaining CSS rules.

```html name=hero.html
<!-- Master CSS Runtime CDN -->
<script src="https://cdn.master.co/css-runtime@rc"></script>
...
<h1 class="**fg:blue** **fg:red:hover** **font:32** **font:40@sm** **font:heavy** **m:10x** **text:center**">
    Hello World
</h1>
```
You can [declare styles](/docs/style-declarations) through the `class` attribute, attaching [selector states](/docs/state-selectors), and even base them on [conditional queries](/docs/conditional-queries). There's no need to write any inline style sheets, including creating external CSS files; just add syntax classes in your HTML.

Whether writing atomic classes `text:center` or [abstract styles](/docs/styles) `btn`, Master CSS does both, and you can choose which one to use based on the characteristics of your project.

### A CSS framework
Whether you're a solo developer or part of a large team, Master CSS can help you build interactive, dynamic, fastest, and beautiful websites.

## Main features
Overview of some of the main features of Master CSS:

### Rendering Modes
Master CSS provides three [rendering modes](/docs/rendering-modes), which you can choose according to project scale and application scenarios to meet your current business requirements.

1. [Progressive Rendering](/docs/rendering-modes/progressive-rendering) scans the rendered HTML ahead of time, server-side or at build time, generates the corresponding CSS rules for each page, and lazy loads the runtime engine to keep track of the dynamic class names.
2. [Runtime Rendering](/docs/rendering-modes/runtime-rendering) observes changes in DOM class names at browser runtime, generates corresponding CSS rules, and injects them into the running style sheet.
3. [Static Extraction](/docs/rendering-modes/static-extraction) integrates build tools to scan project source code, extract class names, generate CSS rules, and write them into a virtual CSS module.

## Performance and benchmarks
Initially, Master CSS was intended to create a CSS engine for production browser runtimes, so we concentrated on performance such as syntax parsing, rule generation, cache sharing, and even browser rendering.

Master CSS doesn't use heavy-duty build tools like [PostCSS](https://postcss.org), [Autoprefixer](https://github.com/postcss/autoprefixer), or any CSS minifiers, and includes these optimizations in the early stages of rule generation.
