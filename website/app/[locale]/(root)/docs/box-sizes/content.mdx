## Using size scale [sr-only]
The box size system is constructed using a [base unit](/docs/configuration#baseunit) of **4 pixels**. For visual consistency, you can use the multiplier unit `x` to apply box sizes with the same scale.
```html
<div class="w:**8x**">32px, width: 2rem</div>
<div class="h:**16x**">64px, height: 4rem</div>
...
```
<Reference />
(x) Unlike traditional, there's no longer a need to define spacing scale one by one.
```js name=master.css.js
- export default {
-     variables: {
-         'box-size': {
-             '1x': 4,
-             '2x': 8,
-             ...,
-             '100x': 400
-         }
-     }
- }
```
By default, Master provides practical container/viewport sizes, `4xs` to `4xl`, and aliases for native box size values.
```html
<div class="**w:md**">1024px, width: 64rem</div>
```

## Basic usage
### Add a box size
Customize your box size tokens by defining [variables](/docs/variables).
```js name=master.css.js
export default {
    variables: {
        'box-size': {
+            desktop: 1280
        }
    }
}
```
Apply the defined `box-size` variables using the inherited rules <Heirs />.
