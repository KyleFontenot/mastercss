## New Configuration API

### `.animations`
You can now customize CSS [animations](/reference/configuration#animations) in `master.css.js`:
```js name=master.css.js
export default {
    animations: {
        fade: {
            '0%': { opacity: 0 },
            'to': { opacity: 1 }
        },
        â€¦
    }
}
```
Apply the `fade` keyframe via [animation syntax](/reference/animation):
```html
<div class="**@fade|.5s|ease**"></div>
```
It only generates the CSS keyframe rule used:
```css
+@keyframes fade {
+    0% {
+        opacity: 0
+    }
+
+    to {
+        opacity: 1
+    }
+}

.\@fade\|\.5s\|ease {
    animation: fade .5s ease
}
```

## Simplified Syntax

### `animation-play-state:` to `@play:`
```html
-<div class="animation-play-state:paused">
-<div class="animation-play-state:running">
+<div class="@play:paused">
+<div class="@play:running">
```
To learn more, check out the [Animaiton Play State](/reference/animation-play-state) documentation.

### `animation-fill-mode:` to `@fill:`
```html
-<div class="animation-fill-mode:forwards">
-<div class="animation-fill-mode:backwards">
-<div class="animation-fill-mode:both">
+<div class="@fill:forwards">
+<div class="@fill:backwards">
+<div class="@fill:both">
```
To learn more, check out the [Animaiton Fill Mode](/reference/animation-fill-mode) documentation.

### `animation-iteration-count:` to `@iteration:`
```html
-<div class="animation-iteration-count:infinite">
+<div class="@fill:infinite">
```
To learn more, check out the [Animaiton Fill Mode](/reference/animation-iteration-count) documentation.

### `animation-direction` values
```html
-<div class="animation-direction:alternate">
-<div class="animation-direction:alternate-reverse">
+<div class="@direction:alt">
+<div class="@direction:alt-reverse">
```
To learn more, check out the [Animaiton Direction](/reference/animation-direction) documentation.
